# Core Operators

The operator layer defines the four fundamental cognitive functions that drive ICA V1.a. Each operator contributes a distinct control signal, and together they form the system’s primary decision‑making and perception engine. Their outputs feed directly into the OS Layer, modulators, and regulators, creating a unified and adaptive cognitive flow.

## Awareness
Awareness tracks the clarity, salience, and structure of incoming information. It evaluates what is present, how stable it is, and how much signal versus noise exists in the current state. This operator anchors perception and provides the baseline for all higher‑order processing.

## Curiosity
Curiosity estimates expected uncertainty reduction. It identifies where exploration is valuable and where additional information would meaningfully improve the system’s internal model. This operator drives adaptive learning and forward movement without destabilizing the architecture.

## Compassion
Compassion applies a danger‑zone penalty function that protects the system from harmful, destabilizing, or high‑risk trajectories. It evaluates potential actions and states for safety, coherence, and long‑term stability. This operator ensures that exploration remains grounded and aligned with system integrity.

## Knowledge
Knowledge integrates updates to the system’s internal models. It consolidates new information, strengthens reliable patterns, and adjusts beliefs based on evidence. This operator maintains continuity, reduces fragmentation, and supports long‑term coherence across the architecture.

## Interaction Pattern
The four operators work as a coordinated set:
- Awareness provides clarity and structure.
- Curiosity identifies valuable uncertainty.
- Compassion constrains unsafe or destabilizing paths.
- Knowledge updates the internal model based on validated information.

Their combined output forms the core cognitive signal that the modulators refine and the regulators stabilize.
