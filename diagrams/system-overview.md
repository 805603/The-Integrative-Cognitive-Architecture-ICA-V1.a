# System Overview Diagram (Text Specification)

This file defines the text‑based specification for the primary system diagram. It provides a clear, minimal blueprint that can later be turned into a visual asset. The goal is to give engineers an immediate, high‑level understanding of ICA V1.a’s structure and signal flow.

## Layered Structure
The architecture is organized into a vertical hierarchy. Each layer feeds into the next, with coherence maintained by the OS Layer and reinforced by regulators and embodied feedback.

- **Suspended‑Aliveness Baseline**  
  Optimal coherence state; the system’s home position.

- **Reflex Arc (Embodied Stability Loop)**  
  Trigger → Physical Cue → Completion Signal → Stabilization → Return to Baseline.

- **Diagnostics**  
  Coherence monitoring, hysteresis, noise detection, transition tracking.

- **Regulators**  
  ICC, UAMC, ASM, PPO, RMO, SLO, Noise Gate, Dampener.

- **Modulators**  
  Gain control, timing, mindfulness, grounding sensitivity, uncertainty weighting.

- **Operators**  
  Awareness, Curiosity, Compassion, Knowledge.

- **OS Layer (Coherence Engine)**  
  Global integrator and stability anchor.

## Signal Flow
- Operators generate primary cognitive signals.  
- Modulators refine intensity, timing, and sensitivity.  
- Regulators intervene when coherence drops or risk increases.  
- Diagnostics monitor stability and inform regulators.  
- Reflex arc provides fast embodied feedback.  
- OS Layer integrates all signals and maintains coherence.  
- Suspended‑aliveness acts as the target state for the entire system.
